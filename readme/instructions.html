AngularJS Phonebook Setup

Introduction

This tutorial will go over the steps necessary to set up an AngularJS application development environment on your machine. We will be using Firebase for application hosting and data storage. This will allow us to better focus on AngularJS application development instead of all the potentially tricky details that come with setting up a server and database on your local machine. We assume that you have access to a basic text editor to work with source files.

Part I: Downloads and Account Creation

The first thing you want to do is head over to nodejs.org/download/ and then download and run the Node.js installer for your operating system of choice (See Figure 1.0). Node.js is a platform built on top of the V8 JavaScript Engine that essentially gives you the ability to run JavaScript programs from the command line. We will be using Node.js for it's npm package manager to download a program - or module in Node.js speak - that we will use to initialize and deploy an AngularJS Firebase application.


(Figure 1.0)

After Node.js is installed, start the command-line and type the following command:

$ npm install -g firebase-tools

This uses the Node.js npm package manager to download Firebase command line tools that will allow you to quickly initialize and deploy an application to Firebase. The -g switch instructs npm to install this module globally on your system, rather than for just a specific application. Note you may need to be root for this command to work properly.

Next goto the Firebase sign-up page to create a Firebase account (See Figure 1.1). Firebase is a free hosting and database service that you can deploy applications to using their simple API.


(Figure 1.1)

After your Firebase account is created you will be redirected to your Firebase dashboard. Make note of the URL shown boxed in red in Figure 1.2 below. This is the link to the datastore where we will store our phonebook contact information - call it the datastore link. We will copy and paste this link into our application in Part II.


(Figure 1.2)

The final thing we need to download is our sample phonebook application. Click here to download it directly. Make sure you remember the download location as we will be accessing it from the command-line in the next step. Regardless of your OS you should have the files in the directory structure similar to Figure 1.3 below.


(Figure 1.3)

Part II: Configuration and Deployment

Next open the controllers.js file located in ~/AngularJS-Phonebook/js/. On line 40 - in the top half of the page - will see a line of the following form:

app.constant('FBURL', 'https://fire-ice-12453.firebaseio.com/');

Change the link to your datastore link (boxed in red in Figure 1.2 above):

app.constant('FBURL', '<your datastore link here>');

Make sure that your data store link contains a forward slash at the end. Once added, save and exit the file.

Next use the command-line and change the directory to the path of where you downloaded the sample phonebook application (~/AngularJS-Phonebook/ on my machine):

$ cd <path to application>

From this location type the following command:

$ firebase init

When prompted, sign-in with your Firebase account. For 'Firebase app:' enter the name of the application that is listed. For the 'Public Directory:' just press enter for (current directory) as we are already in the directory of our application.

Next type:

$ firebase deploy

This will deploy the phonebook application to Firebase.

Finally type:

$ firebase open

This will open the phonebook application in the browser. Note if you are behind the JPL firewall you may need to remove the s from https://.. to properly access the page. To see the contacts click on the 'Initialize Data' link to initialize the data for the application.

Conclusion

That's it! You can now make changes to the AngularJS application. To see changes just type firebase deploy in the applications directory and refresh the application page in the browser. Enjoy.
